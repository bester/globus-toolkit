Myproxy Programs

Preliminaries

The full set of Myproxy programs require either of the following packages be installed:
  • GSI v0.4.0 or later
  • Globus v1.1 or later

Programs

  • myproxy-init
  • myproxy-destroy
  • myproxy-get-delegation
  • myproxy-server

myproxy-init

Creates and sends a delegated proxy to myproxy-server for later retrieval.

SYNOPSIS

myproxy-init [{-h, -u}| {--help, --usage}] [-D | --debug_level level][-v | --version] [-l | --username username] [-c | --cred_lifetime hours] [-t | --proxy_lifetime hours] [-s | --pshost proxy server hostname] [-p | --psport proxy server port #] [-n | --no_passphrase] [-d | --dn_as_username ] [-x | --regex_dn_match] [-X | --match_cn_only] [-a |  --allow_anonymous_retrievers] [-A | --allow_anonymous_renewers]  [-r | --retrievable_by dn ] [-R | --renewable_by dn]

DESCRIPTION

A user will first enter the passphrase necessary to create a proxy certificate. The user is then prompted for a passphrase to protect the delegated proxy on the myproxy-server, unless the no_passphrase argument is specified. The cred_lifetime is the maximum lifetime of the delegated proxy existing on the myproxy-server. The default value is one week. The proxy_lifetime is the maximum lifetime of proxies delegated by the myproxy-server. The default value is 2 hours. If no_passphrase is specified, the proxy can only be retrieved from the myproxy-server with certificate-based authentication for proxy renewal, assuming the myproxy-server allows proxy renewal.

OPTIONS

-h | --help
-D | --debug_level
             Sets debug level. Valid values are 0,1,2
-u | --usage
Displays usage.
-v | --version
Displays version.
-l | --username
Specifies the username for the delegated proxy.
-c | --cred_lifetime
Specifies the maximum lifetime of the delegated proxy on the myproxy-server in hours.
-t | --proxy_lifetime
Specifies the maximum lifetime of proxies delegated by the myproxy-server in hours.
-s | --pshost
Specifies the hostname of myproxy-server.
-p | --psport
Specifies the port number of myproxy-server.
-n | --no_passphrase
Disables passphrase authentication.
-d | --dn_as_username
Use the proxy certificate subject (DN) as the default username, instead of the LOGNAME environment variable.
-x | --regex_dn_match
Specifies DN used by options -r and -R to be matched as a regular expression.
-X | --match_cn_only
Specifies DN used by options -r and -R to be matched with only Common Name (CN).
-a |  --allow_anonymous_retrievers
Allow credentials to be retireved with just username/passphrase.
-A | --allow_anonymous_renewers
Allow credentials to be renewed by any client (not recommended).
-r | --retrievable_by dn
Allow specified entity to retrieve credentials. By default, the argument will be matched against the CN of the client. Use -x to use full regular expression matching instead.
-R | --renewable_by dn
Allow the specified entity to renew credentials. By default, the argument will be matched against the CN of the client. Use -x to use full regular expression matching instead.

EXIT STATUS

0 if the delegated proxy was successfully sent to the myproxy-server.
>0 otherwise (If, for instance, there is a naming conflict between the selected username and existing usernames currently known to the MyProxyServer.

myproxy-destroy

Destroy any existing proxy on the myproxy-server.

SYNOPSIS

myproxy-destroy [{-h, -u} | {--help, --usage}] [-v | --version] [-l | --username username] [-s | --pshost proxy server hostname] [-p | --psport proxy server port #] [-d | --dn_as_username ]

DESCRIPTION

myproxy-destroy removes any delegated proxy certificate from the myproxy-server generated using myproxy-init. The user must have a valid proxy certificate as generated by grid-proxy-init available for myproxy-destroy to use for authentication.

OPTIONS

-h | --help
-u | --usage
Displays usage.
-v | --version
Displays version.
-l | --username
Specifies the username for the delegated proxy.
-s | --pshost
Specifies the hostname of the myproxy-server.
-p | --psport
Specifies the port number of the myproxy-server.
-d | --dn_as_username
Use the proxy certificate subject (DN) as the default username, instead of the LOGNAME environment variable.

EXIT STATUS

0 if the delegated proxy was deleted from the myproxy-server
>0 otherwise

myproxy-get-delegation

Retrieves a delegated credential from the myproxy-server for use by the user and/or web portal.

SYNOPSIS

myproxy-get-delegation [{-h, -u} | {--help, --usage}] [-v | --version] [-l | --username username] [-t | --proxy_lifetime hours] [-o | --out path] [-s | --pshost proxy server hostname] [-p | --psport proxy server port #] [-a | --authorization path to proxy file [-d | --dn_as_username ]

DESCRIPTION

myproxy-get-delegation retrieves a delegated credential for the given username from the myproxy-server. If a valid proxy exists, a delegated proxy will be created for the maximum time dictated by myproxy-init. By default, the delegated proxy is stored in the standard GSI proxy location for that user. The new proxy will be securely delegated using the GSI.

OPTIONS

-h | --help
-u | --usage
Displays usage.
-v | --version
Displays version.
-l | --username
Specifies the username for the delegated proxy.
-proxy_lifetime
Can specify the lifetime of the delegated proxy in hours
-s | --pshost
Specifies the hostname of myproxy-server.
-p | --psport
Specifies the port number of myproxy-server.
-a | --authorization
Specifies a proxy credential file to use for authorization instead of a passphrase, for credential renewal.
-d | --dn_as_username
Use the subject (DN) from the authorization credential as the default username, instead of the LOGNAME environment variable.

EXIT STATUS

0 if the proxy was successfully retrieved from the MyProxyServer
>0 otherwise (If, for instance, there is no available proxy)

myproxy-server

Stores delegated credentials for later retrieval.

SYNOPSIS

myproxy-server [{-h, -u} | {--help, --usage}] [-v | --version][-d | --debug] [-c | --config location of myproxy-server configuration file] [-p | --port proxy server port #] [-s | --storage location of credential storage directory]

DESCRIPTION

myproxy-server is a daemon that runs on a trusted, secure host that manages a database of proxy credentials for use from remote sites. Proxies have a lifetime that is controlled by the myproxy-init program. When a proxy is requested from the myproxy-server, via the myproxy-get-delegation command, further delegation insures that the lifetime of the new proxy may be less than the original to enforce greater security.

A configuration file is responsible for maintaing a list of trusted portals and users that can access this service.

OPTIONS

-h | --help
-D| --debug_level
              Sets debug level. Valid values are 0,1,2
-u | --usage
Displays usage.
-v | --version
Displays version.
-d | --debug
Run in the foreground and log to stderr.
-c | --config
Specifies the location of the myproxy-server configuration file.
-p | --port
Specifies the port number to service requests.
-s | --storage
Specifies the location of the credential storage directory.
